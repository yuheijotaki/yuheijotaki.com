<h2>6 単一コンポーネントによる開発</h2>

<h3>6.2 単一コンポーネントとは</h3>

<blockquote>Vue.js では、上記のような単一ファイルコンポーネントのことを、単一コンポーネントの英語の呼び名の <code>Single File Components</code> の頭文字から、<code>SFC</code>、<code>sfc</code> と略して呼ぶことがあります。また、<code>Vueコンポーネント（Vue Components）</code> と呼ぶことがあります。

</blockquote>

<ul>
<li>単一ファイルコンポーネント</li>
<li>Vue コンポーネント</li>
<li>SFC</li>
</ul>

は同じ。

<h3>6.5 単一コンポーネントの動作を体験する</h3>

Vue CLI は <a href="https://yuheijotaki.hatenablog.com/entry/2018/12/28/025438">ネコ本のとき</a> にやったようなプロジェクト単位の作成でないと使えないと思っていましたが、単一のファイルでも <code>vue serve</code> を使えば使用が可能。  
ちょっと試してみたいときなどにはいいですね。

<code>hello.vue</code> ファイルの中身

```html
<template>
  <p class="message">メッセージ: {{ msg }}</p>
</template>

<script>
  export default {
    data() {
      return {
        msg: "こんにちは"
      };
    }
  };
</script>

<style>
  .message {
    color: red;
  }
</style>
```

コマンドで、<code>hello.vue</code> ファイルの親フォルダに移動して、

```sh
$ vue serve hello.vue --open
```

addon が必要と怒られたので、

```sh
$ yarn global add @vue/cli-service-global
```

eslint がないと怒られたので、

```sh
Error: Cannot find module 'eslint-plugin-vue'
$ npm install --save eslint eslint-plugin-vue@nexeslint-config-vue
```

でもう一度、

```sh
$ vue serve hello.vue --open
```

とすると、<code>http://localhost:8080/</code> の URL で <code>hello.vue</code> が展開される。

<blockquote>ここでは、Vue CLI から、webpack と Vue Loader を使ってバンドルしています。

</blockquote>

<h3>6.6 単一ファイルコンポーネントの機能</h3>

<h4>6.6.1 外部ファイルのインポート</h4>

<blockquote><code>src</code> 属性で以下のように外部ファイルの内容をインポートできます。
パスには、当該の単一ファイルコンポーネントからの相対パスを指定してください。

</blockquote>

```html
<template src="./template.html"></template>
<script src="./script.js"></script>
<style src="./style.css"></style>
```

<h4>6.6.2 スコープ付き CSS</h4>

<blockquote><code>&lt;style&gt;</code> ブロックに <code>scoped</code> 属性を付与することで、その単一ファイルコンポーネント内の要素にのみ適用するカプセル化を実現します。Vue.js の単一ファイルコンポーネントでは、スコープ付き CSS（Scoped CSS）と呼んでいます。

</blockquote>

```html
<style scoped>
  .message {
    color: red;
  }
</style>

<!-- グローバルなCSSも同一ファイルに書ける -->
<style>
  .message {
    font-size: 30px;
  }
</style>
```

とスコープすると、下記のように出力される。（例）

```html
...
<head>
  <style type="text/css">
    .message {
      font-size: 30px;
    }
    .message[data-v-3bcf9374] {
      color: red;
    }
  </style>
</head>
...
```

<h4>6.6.3 CSS モジュール</h4>

CSS モジュールでも名前空間衝突を防止できる。

<code>.vue</code>ファイル

```html
<template>
  <p :class="$style.message">メッセージ: {{ msg }}</p>
</template>

<script>
  export default {
    data() {
      return {
        msg: "こんにちは"
      };
    }
  };
</script>

<style module>
  .message {
    color: blue;
  }
</style>
```

出力結果

```html
<!-- 省略 -->
...
<style type="text/css">
  .hello_message_1mO00 {
    color: blue;
  }
</style>
<!-- 省略 -->
<p class="hello_message_1mO00">メッセージ: こんにちは</p>
...
<!-- 省略 -->
```

<strong>スコープ付き CSS</strong>

接頭辞 <code>data-v-</code> とハッシュ値で構成されたカスタムデータ属性によりカプセル化

<strong>CSS モジュール</strong>

一意なスタイル識別子によってカプセル化。  
スタイル識別子オブジェクトは算出プロパティ <code>$style</code> としてアクセス可能。
