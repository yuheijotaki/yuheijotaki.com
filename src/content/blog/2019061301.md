最近の案件でマークアップに関していろいろ気づきがあったのでメモしておきます。

<h3>見出しタグが入らないエリアは <code>&lt;section&gt;</code> で囲まない</h3>

<strong>良くないと思われる例</strong>

```html
<section>
  <p>テキストです。</p>
</section>
```

<strong>改善例</strong>

```html
<div>
  <p>テキストです。</p>
</div>
```

<h3><code>&lt;section&gt;</code> の中には見出しタグはひとつにする</h3>

<strong>良くないと思われる例</strong>

```html
<section>
  <h1>見出しです。その1</h1>
  <h2>見出しです。その2</h2>
</section>
```

<strong>改善例</strong>

```html
<section>
  <h1>見出しです。その1</h1>
  <section>
    <h2>見出しです。その2</h2>
  </section>
</section>
```

もしくは

```html
<section>
  <h1>
    <span class="title01">見出しです。その1</span>
    <span class="title02">見出しです。その2</span>
  </h1>
</section>
```

<h3>見出しタグは順番にマークアップする</h3>

<strong>良くないと思われる例</strong>

```html
<section>
  <h2>見出しです。その1</h2>
  <h1>見出しです。その2</h1>
</section>
```

<strong>改善例</strong>

```html
<section>
  <h1>見出しです。その1</h1>
  <h2>見出しです。その2</h2>
</section>
```

できない場所は無理やりJSで入れ替えるなども方法としてはあがるようですが、<a href="https://webtan.impress.co.jp/e/2018/04/20/29008">見出しがh1→h2→h3の順でないと、SEOに悪影響あったりします?</a> などを見るとSEO的には関係なさそうです。

<h3><code>&lt;nav&gt;</code> の中には見出しタグがあったほうがよい</h3>

<strong>良くないと思われる例</strong>

```html
<nav>
  <ul>
    <li><a href="#">ナビゲーション</li>
    <li><a href="#">ナビゲーション</li>
    <li><a href="#">ナビゲーション</li>
  </ul>
</nav>
```

<strong>改善例</strong>

```html
<nav>
  <h2>メニュー</h2>
  <ul>
    <li><a href="#">ナビゲーション</li>
    <li><a href="#">ナビゲーション</li>
    <li><a href="#">ナビゲーション</li>
  </ul>
</nav>
```

要素構成上の問題でもありますが、<code>&lt;nav&gt;</code> タグ内に見出しがないと <code>Untitled NAV</code> となってしまいます。  
<a href="https://maboroshi.biz/clearskysource/?p=144">nav要素内に見出しがない |  clear sky source</a>  
ただどうやらエラー認定はされないようでここはあまり気にする必要はないかもしれません。

<h3>まとめ</h3>

宗教論争的な面が多くありそうな話なのでその時々のルールを守って快適に仕事が進むのがいいと思いますが、  
基礎的なルールで間違いていたところがあったので、まとまった時間でもう少し学びたいなと思いました。  
<code>&lt;article&gt;</code> や <code>&lt;aside&gt;</code> あたりもしっかりしようと思います。  
HTMLそれなりに触っては来たけど何も分かっていない気がしてきました。
